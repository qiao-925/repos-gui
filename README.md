# GitHub 仓库批量分类克隆（GUI）

单页 GUI，覆盖**同步 / 批量克隆 / 完整性检查**。默认以可执行文件运行，零命令行（无需了解源码或命令）。

## 🚀 快速开始（仅可执行文件）

- **Windows**：运行 `dist/gh-repos-gui.exe`
- **Linux**：运行 `dist/gh-repos-gui`（如需执行权限，在文件属性中勾选“允许执行”）

> 仅提供可执行文件运行方式；如需源码或构建说明请联系维护者。

## ✅ GUI 覆盖的功能

- **同步**：预览新增公共仓库并写入“未分类”分组
- **批量克隆**：支持增量更新（已存在且完整的仓库会跳过）
- **完整性检查**：仅检查本地仓库（git fsck）
- **失败重试**：自动生成 `failed-repos.txt`，可直接选择该文件重新执行

## ⚙️ 默认参数（可在 GUI 中调整）

- **并行任务数**：5
- **并行连接数**：8
- **检查超时**：30 秒（固定）

## 📄 任务列表文件格式（REPO-GROUPS.md）

```markdown
# GitHub 仓库分组

仓库所有者: qiao-925

## Go-Practice <!-- 397.8号高地 -->
- go-admin
- JavaGuide

## Java-Practice <!-- 597.9号高地 -->
- incubator-seata
- distribute-transaction
```

## 📂 克隆输出目录

默认输出到项目目录上一级的 `repos/` 文件夹，例如：

```
../repos/组名 (高地编号)/仓库名
```

## 📝 备注

- 同步功能仅写入新增仓库，不处理删除/改名/转移。
- 失败列表为 REPO-GROUPS.md 格式，可直接作为任务文件使用。
